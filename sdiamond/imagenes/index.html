Imagenes

De momento solo se pueden usar imagenes en bitmaps. El soporte para otros formatos lo implementare despues. Ahora vamos a tomar una imagen y vamos a colocarla en pantalla. En mi caso voy a utilizar una imagen de 64x64 llamada “imagen.bmp” que colocare en la carpeta del proyecto. Para usar las imagenes vamos a cargar un objeto sdmd::Image. Un detalle que nos importara mas tarde y te servira tener en la parte de atras de la cabeza es que cuando cargemos una imagen vamos a crear una superficie. Ahora sin mas dilacion, pongamos la imagen en pantalla. Luego de crear la ventana vamos a cargar la imagen, para ello necesitamos su ubicacion en el disco (mejor si usas una pocicion relativa al proyecto) y su dimencion.

    //                ("Ubicacion\imagen.bmp", ancho, alto)
    sdmd::Image imagen("imagen.bmp", 64, 64);

Y luego revisamos que se haya cargado correctamente.

    if (!imagen.createdSuccesfully()) {
        sdmd::log_error();
        sdmd::quit();
        return 3;
    }

Ya con la imagen cargada vamos a ponerla en pantalla. Dentro del loop vamos a hacer un blit de la imagen en la superficie de la ventana, y luego actualizamos la ventana. Asi que agregaremos este codigo:

        // blitSurface() pone la imagen segun el miembro posicion de la imagen
        ventana.blitSurface(imagen);
        ventana.updateSurface();

Y veras tu imagen en la pantalla, pero en la esquina superior derecha, yo la quiero ver en otra posicion. Asi que ahora le voy a dar una posicion, digamos en el centro de la pantalla

        // En este caso dar una posicion X y Y va a cambiar sus respectivos valores 
        // en el miembro posicion
        ventana.blitSurface(imagen, 320, 240);
        ventana.updateSurface();

Aqui casi nos sale, pero aun nos falta. Te daras cuenta de que en los ejemplos he estado repitiendo sobre el miembro posicion. Este es un SDL_Rect que almacena la posicion x,y y tambien el ancho(w) y alto(h) de la imagen. La esquina superior izquierda de la imagen quedara en la posicion (x, y), y luego se extiende segun el ancho y alto. Por eso es que aun no nos queda en el centro, asi que vamos a ajustar la posicion ya con todos los detalles que necesitamos.

        imagen.position.x = 320 - imagen.position.w / 2;
        imagen.position.y = 240 - imagen.position.h / 2;
        ventana.blitSurface(imagen);
        ventana.updateSurface();

Y si lo que quieres es mover la imagen, tendras que rellenar la superficie antes de ponerla otravez. Asi que vamos a hacer eso mismo:

        // Es un esquma de colores RGB
        ventana.fillSurface(0x0, 0x0, 0x0);
        imagen.position.x = 320 - imagen.position.w / 2;
        imagen.position.y = 240 - imagen.position.h / 2;
        ventana.blitSurface(imagen);
        ventana.updateSurface();

Y como un ultimo truco, puedes cambiar la dimension de la imagen cambiando w y h en el miembro posicion y la dimension se ajustara automaticamente. Haciendo unas refactorizaciones para tener un codigo mas limpio, terminamos con este codigo:

#define PS_UTIL_TIMER
#define PS_UTIL_LOG

#include <SDL.h> // En linux <SDL2/SDL.h>
#include "ps_utility.hpp"
#include "sDiamond.hpp"

#define SCR_W 640
#define SCR_H 480

int main(int argc, char* argv[]) {
	if (!sdmd::init()) {
		sdmd::log_error();
		sdmd::quit();
		return 1;
	}

	sdmd::Window ventana("Tutorial", SCR_W, SCR_H);
	if (!ventana.createdSuccesfully()) {
		sdmd::log_error();
		sdmd::quit();
		return 2;
	}

	sdmd::Image imagen("imagen.bmp", 64, 64);
	if (!imagen.createdSuccesfully()) {
		sdmd::log_error();
		sdmd::quit();
		return 3;
	}

	sdmd::Event evento;
	bool corriendo = true;
	while(corriendo){
		while(sdmd::Event::poll(evento)) {
			switch(evento.type()) {
			case sdmd::QUIT:
				corriendo = false;
				break;
			default:
				break;
			}
		}

		ventana.fillSurface(0x0, 0x0, 0x0);
		imagen.position.x = SCR_W/2 - imagen.position.w / 2;
		imagen.position.y = SCR_H/2 - imagen.position.h / 2;
		ventana.blitSurface(imagen);
		ventana.updateSurface();
		sdmd::delay(33);
	}


	sdmd::quit();
	return 0;
}
