Estructura del Videojuego

Nuesta ventana esta chilera y todo, pero en realidad aun no estamos haciendo nada. Asi que ahora vamos a responder a las acciones del usuario. A las accioes de mouse y del teclado vamos a aprender a reaccionar luego, ya que ahora tambien quiero introducir un concepto muy importante, y es la estructura que siguen los juegos. Los juegos siguen la siguiente estructura:

    while(elJuegoCorra) {
        1. Leemos los eventos del usuario
        2. Reaccionamos a los mismos
        3. Pasamos las reacciones a la pantalla
        4. Actualizamos la pantalla
    }

Y no solo los videojuegos, sino que todas las aplicaciones con interfaz grafica siguen esta estructura. Asi que vamos a aplicar esta estructura a nuestra app. Primero que nada el evento, vamos a guardar los eventos del usuario en un objeto sdmd::Event. Y tambien vamos a tener una variable que nos diga si la app sigue corriendo o no.

    sdmd::Event evento;
    bool corriendo = true;

Y vamos a dejar los primeros 2 pasos de la estructura, para ello vamos a leer los eventos con la funcion sdmd::Event::poll(). Tenemos que entender que los eventos se guardan en un queue, asi que sdmd::Event::poll() va a devolver verdadero siempre que hayan eventos en el queue. Por ende vamos a ponerlo de la siguiente manera: Mientras haya eventos en el queue reaccionamos a ellos.

    while(corriendo){
        while(sdmd::Event::poll(evento)) {
            // Acciones
        }
    }

Pueden suceder bastantes eventos diferentes, como que se mueva el mouse, se precione el teclado, o se quiera cerrar la ventana. De momento vamos a atender a este ultimo. Para ello revisaremos el tipo de evento que sucedio, y si el tipo de evento nos dice que quiere salir, entonces terminamos la app.

    sdmd::Event evento;
    bool corriendo = true;
    while(corriendo){
        while(sdmd::Event::poll(evento)) {
            switch(evento.type()) {
            case sdmd::QUIT:
                corriendo = false;
                break;
            default:
                break;
            }
        }
    }

No te recomendaria que corrieras la app ahora, nos falta un peque�o detalle. Pero si la corres te recomiendo que abras tu gestor de tareas favorito y mires el uso de cpu que tiene en este momento, llegara cercano al 100%. En este momento el while da mas vueltas que terraplanista debatiendo. Y en realidad no necesitamos que de vueltas tan rapido. Nosotros que de vueltas ajustado al framerate que necesitemos para nuestro juego. Asi que vamos a aprovechar la funcion delay para realentisar un poco las vueltas. En mi caso lo dejo para que corra a 30fps

    sdmd::Event evento;
    bool corriendo = true;
    while(corriendo){
        while(sdmd::Event::poll(evento)) {
            switch(evento.type()) {
            case sdmd::QUIT:
                corriendo = false;
                break;
            default:
                break;
            }
        }

        sdmd::delay(33);
    }

Y ya casi tenemos la estructura basica del juego. Ahora veamos como poner imagenes en pantalla.
