Small Diamond

Esta seria una libreria simple construida sobre SDL para volverla orientada a objetos y para simplificar el proceso de creacion de videojuegos en 2d, ademas de automatizar la limpieza de memoria que requiere SDL. Como un agregado tiene un sistema de registros con macros bastante simple con macros para ayudar a la depuracion y para dar mensajes de error y advertencia de parte de la libreria, ademas de un timer.

Nota: Tanto las utilidades como sDiamond estan bajo la licencia Apache

El tutorial lo terminare cuando baje la carga de otros proyectos, pero lo que falta puedes ver como se hace en la rama “example”

Detalles Previos

Como mencione al principio, este proyecto esta basado en SDL, asi que tendra que incluirlo tambien en su proyecto. En caso de estar en windows y que estes usando visual studio, estas de suerte ya que solo tienes que abrir el proyecto y ya estas listo para empezar a escribir tu juego.

En linux instalas es paquete que corresponda para desarrollar en SDL. Puedes usar cmake, y ya que estamos, borrar include y lib ya que estos solo estarian funcionando en windows . Si deseas utilizar “ps_utility.hpp” puedes siempre y cuando no uses el timer, este es un problema que voy a tratar de resolver en corto.

Y hablando de “ps_utility.hpp” en este header hay dos herramientas, un sistema de registros al que accedes definiendo:

#define PS_UTIL_LOG

Usarias el log de la siguiente manera

    /*
    * Los niveles son:
    * - WL_high (Rojo)
    * - WL_medium (Amarillo)
    * - WL_good (verde)
    * - WL_none 
    * - DB_data (Blanco)
    * - DB_result (Azul)
    */
    LOG(level, message);

Y gracias a la forma en la que esta definido #define LOG(level,message) std::cout<<level<<message<<"\x1b[0m\n" nos permite hacer cosas como:

    LOG(WL_none, "La pelota esta en la posicion (" << x << ", " << y << ")");

Lo que nos permite darle formato a los registros que hagamos. En caso de definir PS_UTIL_LOG tambien recivira mensajes de parte de sDiamod, como cuando creo un objeto, cuando se copia un objeto (si se copia una ventana por ejemplo, ya estamos cometiendo errores), si se cerro correctamente, y tambien recordatorios de cosas que no deve hacer.

La otra herramienta es un timer al que accedes definiendo:

    #define PS_UTIL_TIMER

Este fucniona casi de manera automatica, ya que mide su tiempo de vida. Recuerde que un objeto solo vide dentro de los {}. Por ejemplo:

    int main(void) {
        TIMER(t1, "Toda la app");
        {
            TIMER(t2, "Decir: Hola Mundo");
            std::cout << "Hola Mundo" << std::endl;
        }
        std::cout << "Adios Mundo" << std::endl;
        return 0;
    }

Un ultimo detalle a tomar en cuenta es que el main queda sustituido por un macro, si estas en un IDE pon el siguiente codigo:

    #include<SDL.h> // <SDL2/SDL.h> en linux

    int main(int argc, char *argv[]) {
        return 0;
    }

Podras ver que el main queda como un macro que se sustituye por SDL_main. Esto es porque SDL tiene que hacer cosas antes de que puedas empezar a correr tu juego. Por ende vamos a usar el main de la siguiente manera int main(int argc, char *argv[]) {} para no tener conflictos con el macro de SDL
